# Pattern 1: Arrays & Hashing

### 1. Two Sum
**Difficulty:** Easy  
**LeetCode:** [Two Sum](https://leetcode.com/problems/two-sum/)  
**Pattern trigger:** Need two indices whose values complete a target.  
**Core idea:** Store earlier values in a hash map and ask whether target - current was seen. O(n) time, O(n) space.  
**Algorithm:**
1. Create value -> index map.
2. For each number, compute its complement.
3. Return the saved complement index and current index if found.
4. Insert the current number after checking.
**C++17:**
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> seen;
        for (int i = 0; i < (int)nums.size(); ++i) {
            int need = target - nums[i];
            if (seen.count(need)) return {seen[need], i};
            seen[nums[i]] = i;
        }
        return {};
    }
};
```

### 2. Contains Duplicate
**Difficulty:** Easy  
**LeetCode:** [Contains Duplicate](https://leetcode.com/problems/contains-duplicate/)  
**Pattern trigger:** Need to know whether any value repeats.  
**Core idea:** A hash set detects the first repeated insert immediately. O(n) time, O(n) space.  
**Algorithm:**
1. Create an empty set.
2. Scan each number.
3. If it already exists, return true.
4. Otherwise insert it and finish with false.
**C++17:**
```cpp
class Solution {
public:
    bool containsDuplicate(vector<int>& nums) {
        unordered_set<int> seen;
        for (int x : nums) {
            if (!seen.insert(x).second) return true;
        }
        return false;
    }
};
```

### 3. Valid Anagram
**Difficulty:** Easy  
**LeetCode:** [Valid Anagram](https://leetcode.com/problems/valid-anagram/)  
**Pattern trigger:** Need exact character-frequency equality.  
**Core idea:** For lowercase strings, one fixed array is a perfect counter. O(n) time, O(1) space.  
**Algorithm:**
1. Reject different lengths.
2. Increment counts from s and decrement from t.
3. Scan the 26 counts.
4. All zeros means an anagram.
**C++17:**
```cpp
class Solution {
public:
    bool isAnagram(string s, string t) {
        if (s.size() != t.size()) return false;
        array<int, 26> cnt{};
        for (int i = 0; i < (int)s.size(); ++i) {
            ++cnt[s[i] - 'a'];
            --cnt[t[i] - 'a'];
        }
        for (int x : cnt) {
            if (x != 0) return false;
        }
        return true;
    }
};
```

### 4. Group Anagrams
**Difficulty:** Medium  
**LeetCode:** [Group Anagrams](https://leetcode.com/problems/group-anagrams/)  
**Pattern trigger:** Need to bucket strings by identical character counts.  
**Core idea:** Use a frequency signature as the hash key for each word. O(nk) time, O(nk) space.  
**Algorithm:**
1. Count 26 letters for each string.
2. Serialize the counts with separators.
3. Append the string to that key's bucket.
4. Return all buckets.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> groups;
        for (const string& s : strs) {
            array<int, 26> cnt{};
            for (char c : s) ++cnt[c - 'a'];
            string key;
            for (int x : cnt) key += "#" + to_string(x);
            groups[key].push_back(s);
        }

        vector<vector<string>> ans;
        for (auto& [key, bucket] : groups) ans.push_back(move(bucket));
        return ans;
    }
};
```

### 5. Top K Frequent Elements
**Difficulty:** Medium  
**LeetCode:** [Top K Frequent Elements](https://leetcode.com/problems/top-k-frequent-elements/)  
**Pattern trigger:** Need the most frequent values, not sorted order.  
**Core idea:** Count frequencies, then bucket values by frequency and read high buckets first. O(n) time, O(n) space.  
**Algorithm:**
1. Count each number.
2. Place numbers into buckets indexed by frequency.
3. Scan buckets from high to low.
4. Stop after collecting k values.
**C++17:**
```cpp
class Solution {
public:
    vector<int> topKFrequent(vector<int>& nums, int k) {
        unordered_map<int, int> freq;
        for (int x : nums) ++freq[x];

        vector<vector<int>> buckets(nums.size() + 1);
        for (auto& [num, count] : freq) buckets[count].push_back(num);

        vector<int> ans;
        for (int f = (int)buckets.size() - 1; f >= 0 && (int)ans.size() < k; --f) {
            for (int x : buckets[f]) {
                ans.push_back(x);
                if ((int)ans.size() == k) break;
            }
        }
        return ans;
    }
};
```

### 6. Product of Array Except Self
**Difficulty:** Medium  
**LeetCode:** [Product of Array Except Self](https://leetcode.com/problems/product-of-array-except-self/)  
**Pattern trigger:** Need product excluding self without division.  
**Core idea:** Store prefix products in the answer, then multiply by a running suffix. O(n) time, O(1) extra space.  
**Algorithm:**
1. Fill answer with prefix products before each index.
2. Walk right to left with a suffix product.
3. Multiply answer[i] by the current suffix.
4. Update suffix using nums[i].
**C++17:**
```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        vector<int> ans(n, 1);

        for (int i = 1; i < n; ++i) ans[i] = ans[i - 1] * nums[i - 1];

        int suffix = 1;
        for (int i = n - 1; i >= 0; --i) {
            ans[i] *= suffix;
            suffix *= nums[i];
        }
        return ans;
    }
};
```

### 7. Longest Consecutive Sequence
**Difficulty:** Medium  
**LeetCode:** [Longest Consecutive Sequence](https://leetcode.com/problems/longest-consecutive-sequence/)  
**Pattern trigger:** Need longest run of integer neighbors regardless of order.  
**Core idea:** Only start counting from numbers that have no predecessor. O(n) time, O(n) space.  
**Algorithm:**
1. Put all numbers in a set.
2. For each number, skip it if num - 1 exists.
3. Expand forward while consecutive values exist.
4. Track the longest length.
**C++17:**
```cpp
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        unordered_set<int> st(nums.begin(), nums.end());
        int best = 0;

        for (int x : st) {
            if (st.count(x - 1)) continue;
            int cur = x;
            int len = 1;
            while (st.count(cur + 1)) {
                ++cur;
                ++len;
            }
            best = max(best, len);
        }
        return best;
    }
};
```

# Pattern 2: Two Pointers

### 8. Valid Palindrome
**Difficulty:** Easy  
**LeetCode:** [Valid Palindrome](https://leetcode.com/problems/valid-palindrome/)  
**Pattern trigger:** Need symmetric comparison while ignoring noise.  
**Core idea:** Move two pointers inward and skip non-alphanumeric characters. O(n) time, O(1) space.  
**Algorithm:**
1. Start at both ends.
2. Skip invalid characters on each side.
3. Compare lowercase characters.
4. Move inward until pointers cross.
**C++17:**
```cpp
class Solution {
public:
    bool isPalindrome(string s) {
        int left = 0;
        int right = (int)s.size() - 1;

        while (left < right) {
            while (left < right && !isalnum((unsigned char)s[left])) ++left;
            while (left < right && !isalnum((unsigned char)s[right])) --right;
            if (tolower((unsigned char)s[left]) != tolower((unsigned char)s[right])) return false;
            ++left;
            --right;
        }
        return true;
    }
};
```

### 9. Two Sum II - Input Array Is Sorted
**Difficulty:** Medium  
**LeetCode:** [Two Sum II - Input Array Is Sorted](https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/)  
**Pattern trigger:** Sorted array and pair sum target.  
**Core idea:** Use the sorted order to move the side that makes the sum wrong. O(n) time, O(1) space.  
**Algorithm:**
1. Place pointers at both ends.
2. Compare their sum with target.
3. Move left up if sum is small.
4. Move right down if sum is large.
**C++17:**
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int left = 0;
        int right = (int)numbers.size() - 1;

        while (left < right) {
            int sum = numbers[left] + numbers[right];
            if (sum == target) return {left + 1, right + 1};
            if (sum < target) ++left;
            else --right;
        }
        return {};
    }
};
```

### 10. 3Sum
**Difficulty:** Medium  
**LeetCode:** [3Sum](https://leetcode.com/problems/3sum/)  
**Pattern trigger:** Need unique triplets summing to zero.  
**Core idea:** Sort, fix one value, then solve the remaining two-sum with pointers. O(n^2) time, O(1) extra space.  
**Algorithm:**
1. Sort nums.
2. Fix i and skip duplicate fixed values.
3. Use left/right pointers for -nums[i].
4. After a hit, skip duplicate side values.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        sort(nums.begin(), nums.end());
        vector<vector<int>> ans;
        int n = nums.size();

        for (int i = 0; i < n; ++i) {
            if (i > 0 && nums[i] == nums[i - 1]) continue;
            int left = i + 1;
            int right = n - 1;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == 0) {
                    ans.push_back({nums[i], nums[left], nums[right]});
                    ++left;
                    --right;
                    while (left < right && nums[left] == nums[left - 1]) ++left;
                    while (left < right && nums[right] == nums[right + 1]) --right;
                } else if (sum < 0) {
                    ++left;
                } else {
                    --right;
                }
            }
        }
        return ans;
    }
};
```

### 11. Container With Most Water
**Difficulty:** Medium  
**LeetCode:** [Container With Most Water](https://leetcode.com/problems/container-with-most-water/)  
**Pattern trigger:** Need max area formed by two boundary lines.  
**Core idea:** The smaller wall limits area, so move that pointer inward. O(n) time, O(1) space.  
**Algorithm:**
1. Start with widest container.
2. Update area from current width and smaller height.
3. Move the pointer at the smaller height.
4. Keep the best area seen.
**C++17:**
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int left = 0;
        int right = (int)height.size() - 1;
        int best = 0;

        while (left < right) {
            int h = min(height[left], height[right]);
            best = max(best, h * (right - left));
            if (height[left] < height[right]) ++left;
            else --right;
        }
        return best;
    }
};
```

### 12. Trapping Rain Water
**Difficulty:** Hard  
**LeetCode:** [Trapping Rain Water](https://leetcode.com/problems/trapping-rain-water/)  
**Pattern trigger:** Need water trapped by left and right max boundaries.  
**Core idea:** Process the lower side because its opposite boundary is already sufficient. O(n) time, O(1) space.  
**Algorithm:**
1. Keep two pointers and left/right maxima.
2. Move the side with smaller height.
3. Add max boundary minus current height when positive.
4. Update boundary maxima as you go.
**C++17:**
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int left = 0;
        int right = (int)height.size() - 1;
        int leftMax = 0;
        int rightMax = 0;
        int water = 0;

        while (left < right) {
            if (height[left] < height[right]) {
                leftMax = max(leftMax, height[left]);
                water += leftMax - height[left];
                ++left;
            } else {
                rightMax = max(rightMax, height[right]);
                water += rightMax - height[right];
                --right;
            }
        }
        return water;
    }
};
```

### 13. Best Time to Buy and Sell Stock
**Difficulty:** Easy  
**LeetCode:** [Best Time to Buy and Sell Stock](https://leetcode.com/problems/best-time-to-buy-and-sell-stock/)  
**Pattern trigger:** Need best profit from one buy before one sell.  
**Core idea:** Track the cheapest price before today and sell against it. O(n) time, O(1) space.  
**Algorithm:**
1. Initialize min price and best profit.
2. For each price, compute sell profit.
3. Update best profit.
4. Update the minimum price seen.
**C++17:**
```cpp
class Solution {
public:
    int maxProfit(vector<int>& prices) {
        int low = INT_MAX;
        int best = 0;

        for (int price : prices) {
            low = min(low, price);
            best = max(best, price - low);
        }
        return best;
    }
};
```

### 14. Longest Substring Without Repeating Characters
**Difficulty:** Medium  
**LeetCode:** [Longest Substring Without Repeating Characters](https://leetcode.com/problems/longest-substring-without-repeating-characters/)  
**Pattern trigger:** Need longest window with all unique characters.  
**Core idea:** Remember each character's last index and jump left past repeats. O(n) time, O(1) space.  
**Algorithm:**
1. Store last seen index for each character.
2. Expand right one character at a time.
3. If repeated inside the window, move left after its old index.
4. Update the best window length.
**C++17:**
```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        vector<int> last(256, -1);
        int left = 0;
        int best = 0;

        for (int right = 0; right < (int)s.size(); ++right) {
            unsigned char c = s[right];
            left = max(left, last[c] + 1);
            last[c] = right;
            best = max(best, right - left + 1);
        }
        return best;
    }
};
```

### 15. Longest Repeating Character Replacement
**Difficulty:** Medium  
**LeetCode:** [Longest Repeating Character Replacement](https://leetcode.com/problems/longest-repeating-character-replacement/)  
**Pattern trigger:** Need longest window fixable with at most k replacements.  
**Core idea:** A window is valid when length - maxFrequency <= k. O(n) time, O(1) space.  
**Algorithm:**
1. Expand right and count characters.
2. Track the highest frequency ever inside the window.
3. Shrink while replacements needed exceed k.
4. Record the largest valid window.
**C++17:**
```cpp
class Solution {
public:
    int characterReplacement(string s, int k) {
        array<int, 26> cnt{};
        int left = 0;
        int maxFreq = 0;
        int best = 0;

        for (int right = 0; right < (int)s.size(); ++right) {
            maxFreq = max(maxFreq, ++cnt[s[right] - 'A']);
            while (right - left + 1 - maxFreq > k) {
                --cnt[s[left] - 'A'];
                ++left;
            }
            best = max(best, right - left + 1);
        }
        return best;
    }
};
```

### 16. Permutation in String
**Difficulty:** Medium  
**LeetCode:** [Permutation in String](https://leetcode.com/problems/permutation-in-string/)  
**Pattern trigger:** Need any fixed-length window with same counts as s1.  
**Core idea:** Slide a window of size |s1| and compare 26 counts. O(n) time, O(1) space.  
**Algorithm:**
1. Reject if s1 is longer.
2. Build counts for s1 and first window.
3. Slide one character in and one out.
4. Return true when counts match.
**C++17:**
```cpp
class Solution {
public:
    bool checkInclusion(string s1, string s2) {
        int m = s1.size();
        int n = s2.size();
        if (m > n) return false;

        array<int, 26> need{};
        array<int, 26> win{};
        for (int i = 0; i < m; ++i) {
            ++need[s1[i] - 'a'];
            ++win[s2[i] - 'a'];
        }
        if (need == win) return true;

        for (int right = m; right < n; ++right) {
            ++win[s2[right] - 'a'];
            --win[s2[right - m] - 'a'];
            if (need == win) return true;
        }
        return false;
    }
};
```

### 17. Minimum Window Substring
**Difficulty:** Hard  
**LeetCode:** [Minimum Window Substring](https://leetcode.com/problems/minimum-window-substring/)  
**Pattern trigger:** Need smallest window covering all required character counts.  
**Core idea:** Expand until all chars are covered, then shrink greedily. O(n) time, O(1) space.  
**Algorithm:**
1. Count required characters from t.
2. Expand right and reduce missing count.
3. While covered, update answer and move left.
4. Return the best slice.
**C++17:**
```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        vector<int> need(256, 0);
        for (unsigned char c : t) ++need[c];

        int missing = t.size();
        int bestLen = INT_MAX;
        int bestStart = 0;
        int left = 0;

        for (int right = 0; right < (int)s.size(); ++right) {
            unsigned char c = s[right];
            if (need[c] > 0) --missing;
            --need[c];

            while (missing == 0) {
                if (right - left + 1 < bestLen) {
                    bestLen = right - left + 1;
                    bestStart = left;
                }
                unsigned char d = s[left++];
                ++need[d];
                if (need[d] > 0) ++missing;
            }
        }
        return bestLen == INT_MAX ? "" : s.substr(bestStart, bestLen);
    }
};
```

### 18. Sliding Window Maximum
**Difficulty:** Hard  
**LeetCode:** [Sliding Window Maximum](https://leetcode.com/problems/sliding-window-maximum/)  
**Pattern trigger:** Need maximum of every fixed-size window.  
**Core idea:** Maintain a decreasing deque of candidate indices. O(n) time, O(k) space.  
**Algorithm:**
1. Remove indices outside the window.
2. Pop smaller values from the back.
3. Push the current index.
4. Front holds the max once the first window forms.
**C++17:**
```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        deque<int> dq;
        vector<int> ans;

        for (int i = 0; i < (int)nums.size(); ++i) {
            while (!dq.empty() && dq.front() <= i - k) dq.pop_front();
            while (!dq.empty() && nums[dq.back()] <= nums[i]) dq.pop_back();
            dq.push_back(i);
            if (i >= k - 1) ans.push_back(nums[dq.front()]);
        }
        return ans;
    }
};
```

# Pattern 4: Stack

### 19. Valid Parentheses
**Difficulty:** Easy  
**LeetCode:** [Valid Parentheses](https://leetcode.com/problems/valid-parentheses/)  
**Pattern trigger:** Need match latest opener with current closer.  
**Core idea:** A stack enforces last-opened, first-closed pairing. O(n) time, O(n) space.  
**Algorithm:**
1. Push opening brackets.
2. On a closing bracket, require a matching top.
3. Pop matched pairs.
4. Valid only if stack ends empty.
**C++17:**
```cpp
class Solution {
public:
    bool isValid(string s) {
        stack<char> st;
        unordered_map<char, char> match{{')', '('}, {']', '['}, {'}', '{'}};

        for (char c : s) {
            if (c == '(' || c == '[' || c == '{') {
                st.push(c);
            } else {
                if (st.empty() || st.top() != match[c]) return false;
                st.pop();
            }
        }
        return st.empty();
    }
};
```

### 20. Min Stack
**Difficulty:** Medium  
**LeetCode:** [Min Stack](https://leetcode.com/problems/min-stack/)  
**Pattern trigger:** Need stack top and current minimum in O(1).  
**Core idea:** Store each value with the minimum at that depth. O(1) per operation, O(n) space.  
**Algorithm:**
1. On push, compute min(value, previousMin).
2. Store both value and min.
3. Pop removes both together.
4. top and getMin read stack top fields.
**C++17:**
```cpp
class MinStack {
    stack<pair<int, int>> st;

public:
    MinStack() {}

    void push(int val) {
        int mn = st.empty() ? val : min(val, st.top().second);
        st.push({val, mn});
    }

    void pop() {
        st.pop();
    }

    int top() {
        return st.top().first;
    }

    int getMin() {
        return st.top().second;
    }
};
```

### 21. Evaluate Reverse Polish Notation
**Difficulty:** Medium  
**LeetCode:** [Evaluate Reverse Polish Notation](https://leetcode.com/problems/evaluate-reverse-polish-notation/)  
**Pattern trigger:** Expression where operators consume the last two values.  
**Core idea:** Use a value stack and apply each operator as soon as it appears. O(n) time, O(n) space.  
**Algorithm:**
1. Push numeric tokens.
2. For an operator, pop right then left.
3. Apply the operation.
4. Push the result back.
**C++17:**
```cpp
class Solution {
public:
    int evalRPN(vector<string>& tokens) {
        stack<int> st;

        for (const string& token : tokens) {
            if (token == "+" || token == "-" || token == "*" || token == "/") {
                int b = st.top(); st.pop();
                int a = st.top(); st.pop();
                if (token == "+") st.push(a + b);
                else if (token == "-") st.push(a - b);
                else if (token == "*") st.push(a * b);
                else st.push(a / b);
            } else {
                st.push(stoi(token));
            }
        }
        return st.top();
    }
};
```

### 22. Generate Parentheses
**Difficulty:** Medium  
**LeetCode:** [Generate Parentheses](https://leetcode.com/problems/generate-parentheses/)  
**Pattern trigger:** Need all valid strings with n pairs.  
**Core idea:** Backtrack while never using more ')' than '(' and never exceeding n opens. O(C_n * n) time, O(n) stack.  
**Algorithm:**
1. Track open and closed counts.
2. Add '(' if opens remain.
3. Add ')' only if it stays valid.
4. Save the string at length 2n.
**C++17:**
```cpp
class Solution {
public:
    vector<string> generateParenthesis(int n) {
        vector<string> ans;
        string cur;
        dfs(n, 0, 0, cur, ans);
        return ans;
    }

private:
    void dfs(int n, int open, int close, string& cur, vector<string>& ans) {
        if ((int)cur.size() == 2 * n) {
            ans.push_back(cur);
            return;
        }
        if (open < n) {
            cur.push_back('(');
            dfs(n, open + 1, close, cur, ans);
            cur.pop_back();
        }
        if (close < open) {
            cur.push_back(')');
            dfs(n, open, close + 1, cur, ans);
            cur.pop_back();
        }
    }
};
```

### 23. Daily Temperatures
**Difficulty:** Medium  
**LeetCode:** [Daily Temperatures](https://leetcode.com/problems/daily-temperatures/)  
**Pattern trigger:** Need next greater temperature for each day.  
**Core idea:** A decreasing stack keeps unresolved days until a warmer day appears. O(n) time, O(n) space.  
**Algorithm:**
1. Scan days left to right.
2. Pop colder unresolved days.
3. Set their wait distance to current index difference.
4. Push the current day.
**C++17:**
```cpp
class Solution {
public:
    vector<int> dailyTemperatures(vector<int>& temperatures) {
        vector<int> ans(temperatures.size(), 0);
        stack<int> st;

        for (int i = 0; i < (int)temperatures.size(); ++i) {
            while (!st.empty() && temperatures[i] > temperatures[st.top()]) {
                int j = st.top();
                st.pop();
                ans[j] = i - j;
            }
            st.push(i);
        }
        return ans;
    }
};
```

### 24. Car Fleet
**Difficulty:** Medium  
**LeetCode:** [Car Fleet](https://leetcode.com/problems/car-fleet/)  
**Pattern trigger:** Cars merge if a behind car catches a slower/equal fleet.  
**Core idea:** Sort from closest to target backward and count increasing arrival times. O(n log n) time, O(n) space.  
**Algorithm:**
1. Pair each car's position and time to target.
2. Sort by position descending.
3. A car starts a new fleet if its time exceeds the slowest ahead.
4. Otherwise it joins the fleet ahead.
**C++17:**
```cpp
class Solution {
public:
    int carFleet(int target, vector<int>& position, vector<int>& speed) {
        vector<pair<int, double>> cars;
        for (int i = 0; i < (int)position.size(); ++i) {
            cars.push_back({position[i], (double)(target - position[i]) / speed[i]});
        }
        sort(cars.rbegin(), cars.rend());

        int fleets = 0;
        double slowest = 0.0;
        for (auto& [pos, time] : cars) {
            if (time > slowest) {
                ++fleets;
                slowest = time;
            }
        }
        return fleets;
    }
};
```

### 25. Largest Rectangle in Histogram
**Difficulty:** Hard  
**LeetCode:** [Largest Rectangle in Histogram](https://leetcode.com/problems/largest-rectangle-in-histogram/)  
**Pattern trigger:** Need widest span where each bar is the limiting height.  
**Core idea:** A monotonic stack resolves a bar when a shorter bar fixes its right boundary. O(n) time, O(n) space.  
**Algorithm:**
1. Scan bars plus one sentinel zero.
2. Keep increasing indices on the stack.
3. When current is lower, pop and compute area.
4. Use stack top as left boundary.
**C++17:**
```cpp
class Solution {
public:
    int largestRectangleArea(vector<int>& heights) {
        stack<int> st;
        int best = 0;
        int n = heights.size();

        for (int i = 0; i <= n; ++i) {
            int h = (i == n ? 0 : heights[i]);
            while (!st.empty() && h < heights[st.top()]) {
                int height = heights[st.top()];
                st.pop();
                int left = st.empty() ? -1 : st.top();
                best = max(best, height * (i - left - 1));
            }
            st.push(i);
        }
        return best;
    }
};
```

**LeetCode:** [Binary Search](https://leetcode.com/problems/binary-search/)  
**Pattern trigger:** Sorted array and exact target lookup.  
**Core idea:** Halve the search interval using the middle value. O(log n) time, O(1) space.  
**Algorithm:**
1. Set left and right bounds.
2. Check the middle element.
3. Discard the impossible half.
4. Return -1 if the interval empties.

```cpp
class Solution {

            int mid = left + (right - left) / 2;
            if (nums[mid] == target) return mid;
            if (nums[mid] < target) left = mid + 1;
            else right = mid - 1;
        }
        return -1;
    }
};
```

```cpp
                right = mid - 1;
```

### 32. Median of Two Sorted Arrays
**Difficulty:** Hard  
**LeetCode:** [Median of Two Sorted Arrays](https://leetcode.com/problems/median-of-two-sorted-arrays/)  
**Pattern trigger:** Need median across two sorted arrays without full merge.  
**Core idea:** Binary search the smaller array's partition so left halves contain half the elements. O(log min(m,n)) time.  
**Algorithm:**
1. Always binary search the smaller array.
2. Choose partitions i and j.
3. Check max-left <= min-right on both sides.
4. Move partition until valid, then compute median.
**C++17:**
```cpp
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        if (nums1.size() > nums2.size()) return findMedianSortedArrays(nums2, nums1);

        int m = nums1.size();
        int n = nums2.size();
        int half = (m + n + 1) / 2;
        int left = 0;
        int right = m;

        while (left <= right) {
            int i = left + (right - left) / 2;
            int j = half - i;

            int aLeft = (i == 0 ? INT_MIN : nums1[i - 1]);
            int aRight = (i == m ? INT_MAX : nums1[i]);
            int bLeft = (j == 0 ? INT_MIN : nums2[j - 1]);
            int bRight = (j == n ? INT_MAX : nums2[j]);

            if (aLeft <= bRight && bLeft <= aRight) {
                if ((m + n) % 2) return max(aLeft, bLeft);
                return ((double)max(aLeft, bLeft) + min(aRight, bRight)) / 2.0;
            }
            if (aLeft > bRight) right = i - 1;
            else left = i + 1;
        }
        return 0.0;
    }
};
```

### 39. Add Two Numbers

**Algorithm:**

2. Add current digits plus carry.
3. Append sum % 10.
4. Carry sum / 10 until all input is consumed.

```cpp
                l2 = l2->next;

            carry = sum / 10;
        }
```

### 40. LRU Cache
**Difficulty:** Medium  
**LeetCode:** [LRU Cache](https://leetcode.com/problems/lru-cache/)  
**Pattern trigger:** Need O(1) get/put with least-recent eviction.  
**Core idea:** Hash keys to list nodes; list front is most recent and back is least recent. O(1) per operation.  
**Algorithm:**
1. Store key/value pairs in a doubly linked list.
2. Map keys to list iterators.
3. Move accessed or updated nodes to front.
4. Evict the back when capacity is exceeded.
**C++17:**
```cpp
class LRUCache {
    int cap;
    list<pair<int, int>> items;
    unordered_map<int, list<pair<int, int>>::iterator> pos;

public:
    LRUCache(int capacity) : cap(capacity) {}

    int get(int key) {
        if (!pos.count(key)) return -1;
        items.splice(items.begin(), items, pos[key]);
        return pos[key]->second;
    }

    void put(int key, int value) {
        if (pos.count(key)) {
            pos[key]->second = value;
            items.splice(items.begin(), items, pos[key]);
            return;
        }

        items.push_front({key, value});
        pos[key] = items.begin();

        if ((int)items.size() > cap) {
            int oldKey = items.back().first;
            items.pop_back();
            pos.erase(oldKey);
        }
    }
};
```

### 41. Merge K Sorted Lists

**LeetCode:** [Merge K Sorted Lists](https://leetcode.com/problems/merge-k-sorted-lists/)  
**Pattern trigger:** Need repeatedly choose smallest list head.  
**Core idea:** A min-heap keeps the current smallest node among lists. O(N log k) time, O(k) space.  
**Algorithm:**

2. Pop the smallest node.
3. Append it to the result.
4. Push its next node if present.

```cpp
class Solution {

    ListNode* mergeKLists(vector<ListNode*>& lists) {
        auto cmp = [](ListNode* a, ListNode* b) {
            return a->val > b->val;
        };
        priority_queue<ListNode*, vector<ListNode*>, decltype(cmp)> pq(cmp);

        for (ListNode* node : lists) {
            if (node) pq.push(node);

        ListNode dummy(0);
        ListNode* tail = &dummy;
        while (!pq.empty()) {
            ListNode* node = pq.top();
            pq.pop();
            tail->next = node;
            tail = tail->next;
            if (node->next) pq.push(node->next);
        }
        return dummy.next;
    }
};
```

### 42. Reverse Nodes in K-Group
**Difficulty:** Hard  
**LeetCode:** [Reverse Nodes in K-Group](https://leetcode.com/problems/reverse-nodes-in-k-group/)  
**Pattern trigger:** Need reverse only complete groups of k nodes.  
**Core idea:** Find each kth node, reverse that closed group, and reconnect. O(n) time, O(1) space.  
**Algorithm:**
1. Use a dummy before the list.
2. Find the kth node after groupPrev.
3. Reverse nodes until groupNext.
4. Reconnect and move groupPrev to the old group head.
**C++17:**
```cpp
class Solution {
public:
    ListNode* reverseKGroup(ListNode* head, int k) {
        ListNode dummy(0, head);
        ListNode* groupPrev = &dummy;

        while (true) {
            ListNode* kth = getKth(groupPrev, k);
            if (!kth) break;
            ListNode* groupNext = kth->next;

            ListNode* prev = groupNext;
            ListNode* cur = groupPrev->next;
            while (cur != groupNext) {
                ListNode* next = cur->next;
                cur->next = prev;
                prev = cur;
                cur = next;
            }

            ListNode* oldHead = groupPrev->next;
            groupPrev->next = kth;
            groupPrev = oldHead;
        }
        return dummy.next;
    }

private:
    ListNode* getKth(ListNode* cur, int k) {
        while (cur && k--) cur = cur->next;
        return cur;
    }
};
```

### 59. Last Stone Weight
**Difficulty:** Easy  
**LeetCode:** [Last Stone Weight](https://leetcode.com/problems/last-stone-weight/)  
**Pattern trigger:** Need repeatedly smash two largest values.  
**Core idea:** A max-heap gives the next two heaviest stones efficiently. O(n log n) time, O(n) space.  
**Algorithm:**
1. Push all stones into a max-heap.
2. Pop two largest stones.
3. Push their difference if non-zero.
4. Return remaining stone or zero.
**C++17:**
```cpp
class Solution {
public:
    int lastStoneWeight(vector<int>& stones) {
        priority_queue<int> pq(stones.begin(), stones.end());

            int a = pq.top(); pq.pop();
            int b = pq.top(); pq.pop();
            if (a != b) pq.push(a - b);
        }
        return pq.empty() ? 0 : pq.top();
    }
};
```

### 60. K Closest Points to Origin
**Difficulty:** Medium  
**LeetCode:** [K Closest Points to Origin](https://leetcode.com/problems/k-closest-points-to-origin/)  
**Pattern trigger:** Need k smallest distances, order does not matter.  
**Core idea:** Keep a max-heap of k closest points and evict farther points. O(n log k) time, O(k) space.  
**Algorithm:**
1. Compute squared distance only.
2. Push each point into a max-heap.
3. Pop when heap grows beyond k.
4. Return the heap contents.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<int>> kClosest(vector<vector<int>>& points, int k) {
        auto dist = [](const vector<int>& p) {
            return p[0] * p[0] + p[1] * p[1];
        };

        priority_queue<pair<int, int>> pq;
        for (int i = 0; i < (int)points.size(); ++i) {
            pq.push({dist(points[i]), i});
            if ((int)pq.size() > k) pq.pop();
        }

        vector<vector<int>> ans;
        while (!pq.empty()) {
            ans.push_back(points[pq.top().second]);
            pq.pop();
        }
        return ans;
    }
};
```

### 63. Design Twitter
**Difficulty:** Medium  
**LeetCode:** [Design Twitter](https://leetcode.com/problems/design-twitter/)  
**Pattern trigger:** Need recent tweets from a user and followees.  
**Core idea:** Store tweets per user and merge latest lists with a max-heap. post/follow O(1), feed O(f log f).  
**Algorithm:**
1. Store each user's tweets with increasing time.
2. For feed, seed a heap with each relevant user's latest tweet.
3. Pop newest tweet and push that user's previous tweet.
4. Stop after 10 tweets.
**C++17:**
```cpp
class Twitter {
    int timer = 0;
    unordered_map<int, unordered_set<int>> followees;
    unordered_map<int, vector<pair<int, int>>> tweets;

public:
    Twitter() {}

    void postTweet(int userId, int tweetId) {
        tweets[userId].push_back({timer++, tweetId});
    }

    vector<int> getNewsFeed(int userId) {
        using Item = tuple<int, int, int, int>;
        priority_queue<Item> pq;

        auto addUser = [&](int user) {
            auto& v = tweets[user];
            if (!v.empty()) {
                int i = v.size() - 1;
                pq.push({v[i].first, v[i].second, user, i});
            }
        };

        addUser(userId);
        for (int user : followees[userId]) addUser(user);

        vector<int> feed;
        while (!pq.empty() && feed.size() < 10) {
            auto [time, tweetId, user, idx] = pq.top();
            pq.pop();
            feed.push_back(tweetId);
            if (idx > 0) {
                --idx;
                pq.push({tweets[user][idx].first, tweets[user][idx].second, user, idx});
            }
        }
        return feed;
    }

    void follow(int followerId, int followeeId) {
        if (followerId != followeeId) followees[followerId].insert(followeeId);
    }

    void unfollow(int followerId, int followeeId) {
        followees[followerId].erase(followeeId);
    }
};
```

# Pattern 9: Backtracking

### 65. Subsets
**Difficulty:** Medium  
**LeetCode:** [Subsets](https://leetcode.com/problems/subsets/)  
**Pattern trigger:** Need all choose/skip combinations.  
**Core idea:** Backtracking decides include or exclude at each index. O(n*2^n) time, O(n) stack plus output.  
**Algorithm:**
1. At each index, choose to skip it.
2. Then choose to include it.
3. Save path when index reaches n.
4. Backtrack after include.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<int>> subsets(vector<int>& nums) {
        vector<vector<int>> ans;
        vector<int> path;
        dfs(0, nums, path, ans);
        return ans;
    }

private:
    void dfs(int i, vector<int>& nums, vector<int>& path, vector<vector<int>>& ans) {
        if (i == (int)nums.size()) {
            ans.push_back(path);
            return;
        }
        dfs(i + 1, nums, path, ans);
        path.push_back(nums[i]);
        dfs(i + 1, nums, path, ans);
        path.pop_back();
    }
};
```

### 66. Combination Sum
**Difficulty:** Medium  
**LeetCode:** [Combination Sum](https://leetcode.com/problems/combination-sum/)  
**Pattern trigger:** Need combinations where candidates can repeat.  
**Core idea:** DFS keeps a start index; reusing the same index allows repeats. O(branching^target) time.  
**Algorithm:**
1. Sort candidates for early stopping.
2. If target is zero, save path.
3. Try candidates from start onward.
4. Recurse with same i after choosing.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
        sort(candidates.begin(), candidates.end());
        vector<vector<int>> ans;
        vector<int> path;
        dfs(0, target, candidates, path, ans);
        return ans;
    }

private:
    void dfs(int start, int target, vector<int>& nums, vector<int>& path, vector<vector<int>>& ans) {
        if (target == 0) {
            ans.push_back(path);
            return;
        }
        for (int i = start; i < (int)nums.size() && nums[i] <= target; ++i) {
            path.push_back(nums[i]);
            dfs(i, target - nums[i], nums, path, ans);
            path.pop_back();
        }
    }
};
```

### 67. Permutations
**Difficulty:** Medium  
**LeetCode:** [Permutations](https://leetcode.com/problems/permutations/)  
**Pattern trigger:** Need all orderings of distinct values.  
**Core idea:** Backtrack by choosing each unused number for the next position. O(n*n!) time, O(n) stack plus output.  
**Algorithm:**
1. Track used indices.
2. Choose any unused number.
3. Save when path length is n.
4. Undo choice and continue.
**C++17:**

```cpp
        return ans;

            ans.push_back(path);
            return;
        }
        for (int i = 0; i < (int)nums.size(); ++i) {
            if (used[i]) continue;
            used[i] = true;
            path.push_back(nums[i]);
            dfs(nums, used, path, ans);
            path.pop_back();
            used[i] = false;
        }
    }
};
```

```cpp
        }
        for (int i = start; i < (int)nums.size() && nums[i] <= target; ++i) {
            if (i > start && nums[i] == nums[i - 1]) continue;
            path.push_back(nums[i]);
            dfs(i + 1, target - nums[i], nums, path, ans);
            path.pop_back();
        }
    }
};
```

### 70. Word Search
**Difficulty:** Medium  
**LeetCode:** [Word Search](https://leetcode.com/problems/word-search/)  
**Pattern trigger:** Need trace a word through adjacent cells without reuse.  
**Core idea:** DFS from matching starts and mark cells during the path. O(mn*4^L) time, O(L) stack.  
**Algorithm:**
1. Try every cell as start.
2. DFS only if current char matches word[k].
3. Temporarily mark cell visited.
4. Search four neighbors for k+1.
**C++17:**
```cpp
class Solution {
public:
    bool exist(vector<vector<char>>& board, string word) {
        int m = board.size();
        int n = board[0].size();
        for (int r = 0; r < m; ++r) {
            for (int c = 0; c < n; ++c) {
                if (dfs(board, word, r, c, 0)) return true;
            }
        }
        return false;
    }

private:
    bool dfs(vector<vector<char>>& board, string& word, int r, int c, int k) {
        if (k == (int)word.size()) return true;
        if (r < 0 || c < 0 || r == (int)board.size() || c == (int)board[0].size()) return false;
        if (board[r][c] != word[k]) return false;

        char saved = board[r][c];
        board[r][c] = '#';
        bool found = dfs(board, word, r + 1, c, k + 1) ||
                     dfs(board, word, r - 1, c, k + 1) ||
                     dfs(board, word, r, c + 1, k + 1) ||
                     dfs(board, word, r, c - 1, k + 1);
        board[r][c] = saved;
        return found;
    }
};
```

### 71. Palindrome Partitioning
**Difficulty:** Medium  
**LeetCode:** [Palindrome Partitioning](https://leetcode.com/problems/palindrome-partitioning/)  
**Pattern trigger:** Need split string into only palindromic pieces.  
**Core idea:** Backtrack over end positions and only choose palindromic prefixes. O(n*2^n) time.  
**Algorithm:**
1. Start partitioning at index start.
2. Try every end index.
3. Choose substring only if palindrome.
4. Save path when start reaches n.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<string>> partition(string s) {
        vector<vector<string>> ans;
        vector<string> path;
        dfs(0, s, path, ans);
        return ans;
    }

private:
    void dfs(int start, string& s, vector<string>& path, vector<vector<string>>& ans) {
        if (start == (int)s.size()) {
            ans.push_back(path);
            return;
        }
        for (int end = start; end < (int)s.size(); ++end) {
            if (!isPal(s, start, end)) continue;
            path.push_back(s.substr(start, end - start + 1));
            dfs(end + 1, s, path, ans);
            path.pop_back();
        }
    }

    bool isPal(string& s, int left, int right) {
        while (left < right) {
            if (s[left++] != s[right--]) return false;
        }
        return true;
    }
};
```

### 73. N-Queens
**Difficulty:** Hard  
**LeetCode:** [N-Queens](https://leetcode.com/problems/n-queens/)  
**Pattern trigger:** Need place one queen per row without attacks.  
**Core idea:** Backtrack rows while tracking used columns and diagonals. O(n!) time, O(n^2) output board space.  
**Algorithm:**
1. Place queens row by row.
2. Reject occupied column or diagonal.
3. Mark column and diagonals when choosing.
4. Save board after row n.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<string>> solveNQueens(int n) {
        vector<vector<string>> ans;
        vector<string> board(n, string(n, '.'));
        vector<bool> col(n, false), diag1(2 * n - 1, false), diag2(2 * n - 1, false);
        dfs(0, n, board, col, diag1, diag2, ans);
        return ans;
    }

private:
    void dfs(int r, int n, vector<string>& board, vector<bool>& col,
             vector<bool>& diag1, vector<bool>& diag2, vector<vector<string>>& ans) {
        if (r == n) {
            ans.push_back(board);
            return;
        }
        for (int c = 0; c < n; ++c) {
            int d1 = r - c + n - 1;
            int d2 = r + c;
            if (col[c] || diag1[d1] || diag2[d2]) continue;
            board[r][c] = 'Q';
            col[c] = diag1[d1] = diag2[d2] = true;
            dfs(r + 1, n, board, col, diag1, diag2, ans);
            col[c] = diag1[d1] = diag2[d2] = false;
            board[r][c] = '.';
        }
    }
};
```

### 74. Implement Trie (Prefix Tree)
**Difficulty:** Medium  
**LeetCode:** [Implement Trie (Prefix Tree)](https://leetcode.com/problems/implement-trie-prefix-tree/)  
**Pattern trigger:** Need fast prefix and full-word queries.  
**Core idea:** Each node has 26 child pointers and an end flag. Operations are O(length), space O(total chars).  
**Algorithm:**
1. Walk or create child nodes for insert.
2. Mark final node as word end.
3. For search/prefix, walk existing edges.
4. search also checks end flag.
**C++17:**
```cpp
class Trie {
    array<Trie*, 26> child{};
    bool end = false;

public:
    Trie() {}

    void insert(string word) {
        Trie* node = this;
        for (char c : word) {
            int i = c - 'a';
            if (!node->child[i]) node->child[i] = new Trie();
            node = node->child[i];
        }
        node->end = true;
    }

    bool search(string word) {
        Trie* node = walk(word);
        return node && node->end;
    }

    bool startsWith(string prefix) {
        return walk(prefix) != nullptr;
    }

private:
    Trie* walk(const string& s) {
        Trie* node = this;
        for (char c : s) {
            int i = c - 'a';
            if (!node->child[i]) return nullptr;
            node = node->child[i];
        }
        return node;
    }
};
```

### 75. Design Add and Search Words Data Structure
**Difficulty:** Medium  
**LeetCode:** [Design Add and Search Words Data Structure](https://leetcode.com/problems/design-add-and-search-words-data-structure/)  
**Pattern trigger:** Need trie search with '.' wildcard.  
**Core idea:** Trie handles normal chars; wildcard branches to all children. add O(L), search worst O(26^dots).  
**Algorithm:**
1. Insert words into a trie.
2. Search recursively from root.
3. For a letter, follow one child.
4. For '.', try every non-null child.
**C++17:**
```cpp
class WordDictionary {
    struct Node {
        array<Node*, 26> child{};
        bool end = false;
    };

    Node* root;

public:
    WordDictionary() : root(new Node()) {}

    void addWord(string word) {
        Node* node = root;
        for (char c : word) {
            int i = c - 'a';
            if (!node->child[i]) node->child[i] = new Node();
            node = node->child[i];
        }
        node->end = true;
    }

    bool search(string word) {
        return dfs(word, 0, root);
    }

private:
    bool dfs(const string& word, int i, Node* node) {
        if (!node) return false;
        if (i == (int)word.size()) return node->end;
        if (word[i] != '.') return dfs(word, i + 1, node->child[word[i] - 'a']);

        for (Node* next : node->child) {
            if (dfs(word, i + 1, next)) return true;
        }
        return false;
    }
};
```

### 76. Word Search II
**Difficulty:** Hard  
**LeetCode:** [Word Search II](https://leetcode.com/problems/word-search-ii/)  
**Pattern trigger:** Need find many words on one board.  
**Core idea:** Build a trie to prune DFS paths and clear found words to avoid duplicates. O(mn*4^L) worst time.  
**Algorithm:**
1. Insert all words into a trie.
2. Start DFS from each board cell.
3. Stop if next trie edge is missing.
4. Emit and clear word when a trie terminal is reached.
**C++17:**
```cpp
class Solution {
    struct Node {
        array<Node*, 26> child{};
        string word;
    };

    Node* root = new Node();
    vector<string> ans;

public:
    vector<string> findWords(vector<vector<char>>& board, vector<string>& words) {
        for (const string& word : words) insert(word);
        int m = board.size();
        int n = board[0].size();
        for (int r = 0; r < m; ++r) {
            for (int c = 0; c < n; ++c) {
                dfs(board, r, c, root);
            }
        }
        return ans;
    }

private:
    void insert(const string& word) {
        Node* node = root;
        for (char c : word) {
            int i = c - 'a';
            if (!node->child[i]) node->child[i] = new Node();
            node = node->child[i];
        }
        node->word = word;
    }

    void dfs(vector<vector<char>>& board, int r, int c, Node* node) {
        if (r < 0 || c < 0 || r == (int)board.size() || c == (int)board[0].size()) return;
        char ch = board[r][c];
        if (ch == '#') return;
        Node* next = node->child[ch - 'a'];
        if (!next) return;

        if (!next->word.empty()) {
            ans.push_back(next->word);
            next->word.clear();
        }

        board[r][c] = '#';
        dfs(board, r + 1, c, next);
        dfs(board, r - 1, c, next);
        dfs(board, r, c + 1, next);
        dfs(board, r, c - 1, next);
        board[r][c] = ch;
    }
};
```

```cpp
class Solution {
```

```cpp
public:
    Node* cloneGraph(Node* node) {
```

**Core idea:** DFS returns area while marking visited land. O(mn) time, O(mn) stack worst case.  
**Algorithm:**
1. Scan each cell.
2. DFS only from land.
3. Mark visited cells as water.

```cpp
class Solution {

    int maxAreaOfIsland(vector<vector<int>>& grid) {

        for (int r = 0; r < (int)grid.size(); ++r) {
```

### 80. Pacific Atlantic Water Flow
**Difficulty:** Medium  
**LeetCode:** [Pacific Atlantic Water Flow](https://leetcode.com/problems/pacific-atlantic-water-flow/)  
**Pattern trigger:** Need cells that can reach both oceans by downhill flow.  
**Core idea:** Reverse the flow: DFS from each ocean into non-decreasing heights. O(mn) time, O(mn) space.  
**Algorithm:**
1. Mark cells reachable from Pacific borders.
2. Mark cells reachable from Atlantic borders.
3. Move only to height >= previous height.
4. Return cells marked by both searches.
**C++17:**
```cpp
class Solution {
public:
    vector<vector<int>> pacificAtlantic(vector<vector<int>>& heights) {
        int m = heights.size();
        int n = heights[0].size();
        vector<vector<int>> pac(m, vector<int>(n, 0));
        vector<vector<int>> atl(m, vector<int>(n, 0));

        for (int r = 0; r < m; ++r) {
            dfs(heights, pac, r, 0, INT_MIN);
            dfs(heights, atl, r, n - 1, INT_MIN);
        }
        for (int c = 0; c < n; ++c) {
            dfs(heights, pac, 0, c, INT_MIN);
            dfs(heights, atl, m - 1, c, INT_MIN);
        }

        vector<vector<int>> ans;
        for (int r = 0; r < m; ++r) {
            for (int c = 0; c < n; ++c) {
                if (pac[r][c] && atl[r][c]) ans.push_back({r, c});
            }
        }
        return ans;
    }

private:
    void dfs(vector<vector<int>>& h, vector<vector<int>>& seen, int r, int c, int prev) {
        if (r < 0 || c < 0 || r == (int)h.size() || c == (int)h[0].size()) return;
        if (seen[r][c] || h[r][c] < prev) return;
        seen[r][c] = 1;
        dfs(h, seen, r + 1, c, h[r][c]);
        dfs(h, seen, r - 1, c, h[r][c]);
        dfs(h, seen, r, c + 1, h[r][c]);
        dfs(h, seen, r, c - 1, h[r][c]);
    }
```

### 82. Rotting Oranges
**Difficulty:** Medium  
**LeetCode:** [Rotting Oranges](https://leetcode.com/problems/rotting-oranges/)  
**Pattern trigger:** Need minimum minutes for infection to spread across a grid.  
**Core idea:** Multi-source BFS spreads rot minute by minute. O(mn) time, O(mn) space.  
**Algorithm:**
1. Queue all rotten oranges and count fresh ones.
2. Process BFS level by level.
3. Rot adjacent fresh oranges.
4. Return minutes if no fresh remain, else -1.
**C++17:**
```cpp
class Solution {
public:
    int orangesRotting(vector<vector<int>>& grid) {
        int m = grid.size();
        int n = grid[0].size();
        queue<pair<int, int>> q;
        int fresh = 0;

        for (int r = 0; r < m; ++r) {
            for (int c = 0; c < n; ++c) {
                if (grid[r][c] == 2) q.push({r, c});
                else if (grid[r][c] == 1) ++fresh;
            }
        }

        int minutes = 0;
        int dirs[5] = {1, 0, -1, 0, 1};
        while (!q.empty() && fresh > 0) {
            int sz = q.size();
            while (sz--) {
                auto [r, c] = q.front();
                q.pop();
                for (int d = 0; d < 4; ++d) {
                    int nr = r + dirs[d];
                    int nc = c + dirs[d + 1];
                    if (nr < 0 || nc < 0 || nr == m || nc == n || grid[nr][nc] != 1) continue;
                    grid[nr][nc] = 2;
                    --fresh;
                    q.push({nr, nc});
                }
            }
            ++minutes;
        }
        return fresh == 0 ? minutes : -1;
    }
};
```

### 83. Course Schedule
**Difficulty:** Medium  
**LeetCode:** [Course Schedule](https://leetcode.com/problems/course-schedule/)  
**Pattern trigger:** Need know whether directed prerequisites contain a cycle.  
**Core idea:** Kahn topological BFS succeeds only if all courses become zero-indegree. O(V+E) time, O(V+E) space.  
**Algorithm:**
1. Build graph prereq -> course and indegrees.
2. Queue all zero-indegree courses.
3. Remove courses and decrement neighbors.
4. All courses removed means possible.
**C++17:**

```cpp
class Solution {
public:
    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        vector<vector<int>> graph(numCourses);
        vector<int> indeg(numCourses, 0);

        for (auto& p : prerequisites) {
            graph[p[1]].push_back(p[0]);
            ++indeg[p[0]];
        }

        queue<int> q;
        for (int i = 0; i < numCourses; ++i) {
            if (indeg[i] == 0) q.push(i);
        }

        int done = 0;
        while (!q.empty()) {
            int course = q.front();
            q.pop();
            ++done;
            for (int next : graph[course]) {
                if (--indeg[next] == 0) q.push(next);
            }
        }
        return done == numCourses;
    }
};
```

### 85. Word Ladder
**Difficulty:** Hard  
**LeetCode:** [Word Ladder](https://leetcode.com/problems/word-ladder/)  
**Pattern trigger:** Need shortest transformations with one-letter edges.  
**Core idea:** BFS over words, generating neighbors by changing each character. O(N*L*26) time, O(N) space.  
**Algorithm:**
1. Put wordList into a set.
2. BFS from beginWord by levels.
3. Generate all one-letter mutations.
4. Erase visited words when queued.
**C++17:**
```cpp
class Solution {
public:
    int ladderLength(string beginWord, string endWord, vector<string>& wordList) {
        unordered_set<string> words(wordList.begin(), wordList.end());
        if (!words.count(endWord)) return 0;

        queue<pair<string, int>> q;
        q.push({beginWord, 1});
        words.erase(beginWord);

        while (!q.empty()) {
            auto [word, steps] = q.front();
            q.pop();
            if (word == endWord) return steps;

            for (int i = 0; i < (int)word.size(); ++i) {
                string next = word;
                for (char c = 'a'; c <= 'z'; ++c) {
                    next[i] = c;
                    if (words.count(next)) {
                        words.erase(next);
                        q.push({next, steps + 1});
                    }
                }
            }
        }
        return 0;
    }
};
```

### 86. Climbing Stairs

**Pattern trigger:** Ways to reach step n depend on previous two steps.  
**Core idea:** Classic Fibonacci DP with two variables. O(n) time, O(1) space.  
**Algorithm:**
1. Handle n <= 2 directly.
2. Track ways to previous two steps.
3. Current ways are their sum.
4. Shift variables forward.
**C++17:**
```cpp
class Solution {
public:
    int climbStairs(int n) {
        if (n <= 2) return n;
        int a = 1;
        int b = 2;
        for (int step = 3; step <= n; ++step) {
            int c = a + b;
            a = b;
            b = c;
        }
        return b;
    }
};
```

### 87. Min Cost Climbing Stairs
**Difficulty:** Easy  
**LeetCode:** [Min Cost Climbing Stairs](https://leetcode.com/problems/min-cost-climbing-stairs/)  
**Pattern trigger:** Minimum cost to reach current step from one or two behind.  
**Core idea:** Rolling DP computes cost to stand on the top. O(n) time, O(1) space.  
**Algorithm:**
1. Let dp[i] be min cost to reach step i.
2. Transition from i-1 or i-2 plus their costs.
3. Roll two previous states.
4. Return dp[n].
**C++17:**
```cpp
class Solution {
public:
    int minCostClimbingStairs(vector<int>& cost) {
        int prev2 = 0;
        int prev1 = 0;

        for (int i = 2; i <= (int)cost.size(); ++i) {
            int cur = min(prev1 + cost[i - 1], prev2 + cost[i - 2]);
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
};
```

### 88. House Robber
**Difficulty:** Medium  
**LeetCode:** [House Robber](https://leetcode.com/problems/house-robber/)  
**Pattern trigger:** Need max sum with no adjacent houses.  
**Core idea:** At each house choose rob current plus dp[i-2] or skip with dp[i-1]. O(n) time, O(1) space.  
**Algorithm:**
1. Track best excluding previous house.
2. Track best including up to previous house.
3. For each value, compute max(skip, take).
4. Shift rolling states.
**C++17:**
```cpp
class Solution {
public:
    int rob(vector<int>& nums) {
        int prev2 = 0;
        int prev1 = 0;
        for (int x : nums) {
            int cur = max(prev1, prev2 + x);
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
};
```

### 89. House Robber II
**Difficulty:** Medium  
**LeetCode:** [House Robber II](https://leetcode.com/problems/house-robber-ii/)  
**Pattern trigger:** Circular adjacency means first and last cannot both be robbed.  
**Core idea:** Solve linear robber twice: exclude first or exclude last. O(n) time, O(1) space.  
**Algorithm:**
1. If one house, return it.
2. Rob range [0, n-2].
3. Rob range [1, n-1].
4. Return the larger result.
**C++17:**
```cpp
class Solution {
public:
    int rob(vector<int>& nums) {
        int n = nums.size();
        if (n == 1) return nums[0];
        return max(robRange(nums, 0, n - 2), robRange(nums, 1, n - 1));
    }

private:
    int robRange(vector<int>& nums, int left, int right) {
        int prev2 = 0;
        int prev1 = 0;
        for (int i = left; i <= right; ++i) {
            int cur = max(prev1, prev2 + nums[i]);
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
};
```

### 90. Longest Palindromic Substring
**Difficulty:** Medium  
**LeetCode:** [Longest Palindromic Substring](https://leetcode.com/problems/longest-palindromic-substring/)  
**Pattern trigger:** Need longest center-expanding symmetric substring.  
**Core idea:** Every palindrome has a one-char or two-char center. O(n^2) time, O(1) space.  
**Algorithm:**
1. Try each index as odd center.
2. Try each gap as even center.
3. Expand while characters match.
4. Track best start and length.
**C++17:**
```cpp
class Solution {
public:
    string longestPalindrome(string s) {
        int start = 0;
        int best = 0;

        for (int i = 0; i < (int)s.size(); ++i) {
            expand(s, i, i, start, best);
            expand(s, i, i + 1, start, best);
        }
        return s.substr(start, best);
    }

private:
    void expand(string& s, int left, int right, int& start, int& best) {
        while (left >= 0 && right < (int)s.size() && s[left] == s[right]) {
            --left;
            ++right;
        }
        int len = right - left - 1;
        if (len > best) {
            best = len;
            start = left + 1;
        }
    }
};
```

### 91. Palindromic Substrings
**Difficulty:** Medium  
**LeetCode:** [Palindromic Substrings](https://leetcode.com/problems/palindromic-substrings/)  
**Pattern trigger:** Need count all palindromes, not just longest.  
**Core idea:** Expand around every odd and even center and count matches. O(n^2) time, O(1) space.  
**Algorithm:**
1. Loop through every index.
2. Count odd-center palindromes.
3. Count even-center palindromes.
4. Add every successful expansion.
**C++17:**
```cpp
class Solution {
public:
    int countSubstrings(string s) {
        int ans = 0;
        for (int i = 0; i < (int)s.size(); ++i) {
            ans += count(s, i, i);
            ans += count(s, i, i + 1);
        }
        return ans;
    }

private:
    int count(string& s, int left, int right) {
        int total = 0;
        while (left >= 0 && right < (int)s.size() && s[left] == s[right]) {
            ++total;
            --left;
            ++right;
        }
        return total;
    }
};
```

### 92. Decode Ways
**Difficulty:** Medium  
**LeetCode:** [Decode Ways](https://leetcode.com/problems/decode-ways/)  
**Pattern trigger:** Ways depend on valid one-digit and two-digit endings.  
**Core idea:** Rolling DP adds ways from i-1 and i-2 when the ending code is valid. O(n) time, O(1) space.  
**Algorithm:**
1. Reject strings starting with 0.
2. For each position, test one-digit validity.
3. Test two-digit validity from 10 to 26.
4. Roll previous two counts.
**C++17:**
```cpp
class Solution {
public:
    int numDecodings(string s) {
        if (s.empty() || s[0] == '0') return 0;
        int prev2 = 1;
        int prev1 = 1;

        for (int i = 1; i < (int)s.size(); ++i) {
            int cur = 0;
            if (s[i] != '0') cur += prev1;
            int two = (s[i - 1] - '0') * 10 + (s[i] - '0');
            if (two >= 10 && two <= 26) cur += prev2;
            prev2 = prev1;
            prev1 = cur;
        }
        return prev1;
    }
};
```

### 93. Coin Change
**Difficulty:** Medium  
**LeetCode:** [Coin Change](https://leetcode.com/problems/coin-change/)  
**Pattern trigger:** Need minimum number of coins for each amount.  
**Core idea:** Bottom-up unbounded knapsack relaxes dp[amount] using each coin. O(amount*coins) time, O(amount) space.  
**Algorithm:**
1. Initialize dp[0] = 0 and others INF.
2. For every amount, try each coin.
3. Relax from dp[amount - coin] + 1.
4. Return -1 if target remains INF.
**C++17:**
```cpp
class Solution {
public:
    int coinChange(vector<int>& coins, int amount) {
        const int INF = amount + 1;
        vector<int> dp(amount + 1, INF);
        dp[0] = 0;

        for (int a = 1; a <= amount; ++a) {
            for (int coin : coins) {
                if (a >= coin) dp[a] = min(dp[a], dp[a - coin] + 1);
            }
        }
        return dp[amount] == INF ? -1 : dp[amount];
    }
};
```

### 94. Maximum Product Subarray
**Difficulty:** Medium  
**LeetCode:** [Maximum Product Subarray](https://leetcode.com/problems/maximum-product-subarray/)  
**Pattern trigger:** Negative values can turn smallest product into largest.  
**Core idea:** Track max and min product ending at each index. O(n) time, O(1) space.  
**Algorithm:**
1. Initialize max, min, and best with nums[0].
2. Swap max/min when current is negative.
3. Extend or restart at current value.
4. Update global best.
**C++17:**
```cpp
class Solution {
public:
    int maxProduct(vector<int>& nums) {
        int hi = nums[0];
        int lo = nums[0];
        int best = nums[0];

        for (int i = 1; i < (int)nums.size(); ++i) {
            if (nums[i] < 0) swap(hi, lo);
            hi = max(nums[i], hi * nums[i]);
            lo = min(nums[i], lo * nums[i]);
            best = max(best, hi);
        }
        return best;
    }
};
```

### 95. Word Break
**Difficulty:** Medium  
**LeetCode:** [Word Break](https://leetcode.com/problems/word-break/)  
**Pattern trigger:** Need know if prefix can be segmented by dictionary words.  
**Core idea:** DP marks reachable cut positions and tests dictionary substrings. O(n^3) worst time with substr, O(n) space.  
**Algorithm:**
1. Put dictionary words in a set.
2. Let dp[i] mean s[0:i] can be segmented.
3. For each i, try every previous cut j.
4. Set dp[i] when dp[j] and s[j:i] is a word.
**C++17:**
```cpp
class Solution {
public:
    bool wordBreak(string s, vector<string>& wordDict) {
        unordered_set<string> dict(wordDict.begin(), wordDict.end());
        vector<bool> dp(s.size() + 1, false);
        dp[0] = true;

        for (int i = 1; i <= (int)s.size(); ++i) {
            for (int j = 0; j < i; ++j) {
                if (dp[j] && dict.count(s.substr(j, i - j))) {
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[s.size()];
    }
};
```

### 96. Longest Increasing Subsequence
**Difficulty:** Medium  
**LeetCode:** [Longest Increasing Subsequence](https://leetcode.com/problems/longest-increasing-subsequence/)  
**Pattern trigger:** Need length of best increasing subsequence.  
**Core idea:** Patience sorting keeps the smallest possible tail for each length. O(n log n) time, O(n) space.  
**Algorithm:**
1. Keep tails sorted.
2. For each number, find first tail >= number.
3. Replace it or append if none exists.
4. tails size is LIS length.
**C++17:**
```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        vector<int> tails;
        for (int x : nums) {
            auto it = lower_bound(tails.begin(), tails.end(), x);
            if (it == tails.end()) tails.push_back(x);
            else *it = x;
        }
        return tails.size();
    }
};
```

# Pattern 13: Dynamic Programming - 2D

### 97. Unique Paths
**Difficulty:** Medium  
**LeetCode:** [Unique Paths](https://leetcode.com/problems/unique-paths/)  
**Pattern trigger:** Grid paths move only right or down.  
**Core idea:** 1D DP accumulates paths from left and above. O(mn) time, O(n) space.  
**Algorithm:**
1. Initialize first row as all ones.
2. For each later row, scan columns left to right.
3. dp[c] already holds paths from above.
4. Add dp[c-1] for paths from left.
**C++17:**
```cpp
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<int> dp(n, 1);
        for (int r = 1; r < m; ++r) {
            for (int c = 1; c < n; ++c) {
                dp[c] += dp[c - 1];
            }
        }
        return dp[n - 1];
    }
};
```

### 98. Longest Common Subsequence
**Difficulty:** Medium  
**LeetCode:** [Longest Common Subsequence](https://leetcode.com/problems/longest-common-subsequence/)  
**Pattern trigger:** Need longest sequence preserving order in both strings.  
**Core idea:** DP compares prefixes; matching chars extend the diagonal. O(mn) time, O(n) space.  
**Algorithm:**
1. Use previous and current DP rows.
2. If chars match, take prev[j-1] + 1.
3. Otherwise take max of left/up.
4. Return last cell.
**C++17:**
```cpp
class Solution {
public:
    int longestCommonSubsequence(string text1, string text2) {
        int m = text1.size();
        int n = text2.size();
        vector<int> prev(n + 1, 0), cur(n + 1, 0);

        for (int i = 1; i <= m; ++i) {
            for (int j = 1; j <= n; ++j) {
                if (text1[i - 1] == text2[j - 1]) cur[j] = prev[j - 1] + 1;
                else cur[j] = max(prev[j], cur[j - 1]);
            }
            swap(prev, cur);
            fill(cur.begin(), cur.end(), 0);
        }
        return prev[n];
    }
};
```

### 99. Edit Distance
**Difficulty:** Medium  
**LeetCode:** [Edit Distance](https://leetcode.com/problems/edit-distance/)  
**Pattern trigger:** Need minimum insert/delete/replace operations between prefixes.  
**Core idea:** DP over prefixes chooses replace, delete, or insert. O(mn) time, O(n) space.  
**Algorithm:**
1. Initialize converting empty prefix to word2 prefixes.
2. For each word1 prefix, set delete-to-empty cost.
3. If chars match, copy diagonal.
4. Else take 1 + min(replace, delete, insert).
**C++17:**
```cpp
class Solution {
public:
    int minDistance(string word1, string word2) {
        int m = word1.size();
        int n = word2.size();
        vector<int> dp(n + 1);
        iota(dp.begin(), dp.end(), 0);

        for (int i = 1; i <= m; ++i) {
            int prevDiag = dp[0];
            dp[0] = i;
            for (int j = 1; j <= n; ++j) {
                int saved = dp[j];
                if (word1[i - 1] == word2[j - 1]) {
                    dp[j] = prevDiag;
                } else {
                    dp[j] = 1 + min({prevDiag, dp[j], dp[j - 1]});
                }
                prevDiag = saved;
            }
        }
        return dp[n];
    }
};
```

### 100. Burst Balloons
**Difficulty:** Hard  
**LeetCode:** [Burst Balloons](https://leetcode.com/problems/burst-balloons/)  
**Pattern trigger:** Need max coins when bursting order changes neighbors.  
**Core idea:** Interval DP chooses the last balloon burst inside each interval. O(n^3) time, O(n^2) space.  
**Algorithm:**
1. Pad nums with 1 on both ends.
2. Let dp[l][r] be best for balloons l..r.
3. Try each k as the last burst in interval.
4. Build answers by increasing interval length.
**C++17:**
```cpp
class Solution {
public:
    int maxCoins(vector<int>& nums) {
        int n = nums.size();
        vector<int> a(n + 2, 1);
        for (int i = 0; i < n; ++i) a[i + 1] = nums[i];

        vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));
        for (int len = 1; len <= n; ++len) {
            for (int left = 1; left + len - 1 <= n; ++left) {
                int right = left + len - 1;
                for (int k = left; k <= right; ++k) {
                    dp[left][right] = max(dp[left][right],
                        dp[left][k - 1] + a[left - 1] * a[k] * a[right + 1] + dp[k + 1][right]);
                }
            }
        }
        return dp[1][n];
    }
};
```
